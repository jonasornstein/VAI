# Proposal generation strategies

Three modes for generating betting system proposals.

| Mode | Doc | Primary owner |
|------|-----|---------------|
| Random | [random.md](./random.md) | Povl |
| Expert | [expert.md](./expert.md) | Nisse (+ Povl for cost constraints) |
| Quantitative | [quantitative.md](./quantitative.md) | Povl |

All modes share:

- Input: race card (8 legs for V85)
- Output: proposal artifact per `.grok/rules/03-documentation.md`
- Validation: combination count and SEK cost
